🔧 实施方案（7 步完成）

第一步：准备 Mac Mini

# 推荐配置
- Mac Mini M4 Pro（24-48GB RAM）
- 512GB SSD 或更大（Xcode + DerivedData 占用大）
- 安装 macOS 最新稳定版
- 确保网络稳定（有线网络推荐）

第二步：在 GitHub 仓库添加 Runner

# 1. 进入 GitHub 仓库 → Settings → Actions → Runners
# 2. 点击 "New self-hosted runner"
# 3. 选择：macOS / ARM64 / x64
# 4. 下载 Runner 包（会生成下载 + 配置命令）

第三步：配置 Runner

# 在 Mac Mini 上执行（下载后）
# 解压
unzip actions-runner-osx-arm64-*.zip

# 进入目录
cd actions-runner-osx-arm64-*

# 运行配置（替换 YOUR_TOKEN 和 YOUR_URL）
./config.sh --url https://github.com/YOUR_ORG/YOUR_REPO --token YOUR_TOKEN

# 配置示例输出
√ Connected to GitHub
# Runner registration is complete!

第四步：配置 Runner Labels

# 给 Runner 打标签，方便 workflow 选择
./config.sh --labels mac-mini-m4,ios,production

# 推荐标签：
# - mac-mini-m4（硬件标识）
# - ios / macos（平台）
# - main / staging（分支环境）
# - xcode-15-3（Xcode 版本）

第五步：设置为服务（后台运行）

# 安装为 launchd 服务，开机自启
sudo ./svc.sh install

# 启动服务
./svc.sh start

# 查看状态
./svc.sh status

# 查看日志
tail -f _diag/*.log

第六步：配置 Workflow 使用自托管 Runner

# .github/workflows/ci.yml
name: iOS CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  build:
    # 使用自托管 runner
    runs-on: [self-hosted, mac-mini-m4, ios]
    
    steps:
      - name: Checkout
        uses: actions/checkout@v4
      
      - name: Select Xcode
        run: sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
      
      - name: Build
        run: |
          xcodebuild -project MyApp.xcodeproj \
            -scheme MyApp \
            -configuration Debug \
            build CODE_SIGN_IDENTITY="" CODE_SIGNING_REQUIRED=NO
      
      - name: Run Tests
        run: |
          xcodebuild test \
            -project MyApp.xcodeproj \
            -scheme MyApp \
            -destination 'platform=iOS Simulator,name=iPhone 16 Pro'

  # GitHub 托管的 Job（不受影响）
  lint:
    runs-on: ubuntu-latest
    steps:
      - run: echo "Linux Job 走 GitHub 托管"

第七步：安全配置

# 1. Runner 只读权限：Settings → Actions → Runner settings
#    勾选 "Disable any SCM functionality"

# 2. 设置 IP 白名单（企业用）
#    Settings → Actions → Add IP allowlist

# 3. 定期更新 Runner 版本
./run.sh --version && ./svc.sh stop && ./svc.sh start

💡 进阶技巧

  • 多 Repo 共享 Runner：一台 Mac Mini 可以同时服务多个仓库，只需一个 Runner 注册 token 即可
  • 持久化缓存：DerivedData 和 CocoaPods 缓存在本地，第二次构建速度提升 3-5 倍
  • 并发数控制：macOS Runner 推荐同时跑 4-8 个 Job，避免内存不足
  • 分离构建池：main 分支用稳定 Runner，PR 用临时 Runner，防止污染
  • Runner Group：在 GitHub Org 设置 Runner Group，控制不同团队的访问权限

⚠️ 常见问题

  • Runner 离线：检查 ./svc.sh status，确保 runner.sh 在运行；检查 GitHub 仓库 Settings → Actions → Runners 的连接状态
  • Xcode 版本冲突：多个项目需要不同 Xcode 版本时，用 sudo xcode-select -s /path/to/Xcode.app 切换
  • 内存不足：减少并发 Job 数（4GB RAM / Job），或升级到 48GB 配置
  • 磁盘空间：Xcode + DerivedData 可占用 50-100GB，定期清理：rm -rf ~/Library/Developer/Xcode/DerivedData/*
  • 自签名证书：iOS 构建需要 Apple Developer 证书，在 Keychain 中正确配置签名证书
