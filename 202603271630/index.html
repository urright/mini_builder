🔧 实施方案

第一步：安装 Ollama

# 方法1: 官网下载 (推荐)
# 访问 https://ollama.com/download
# 下载 macOS 版本并安装

# 方法2: Homebrew
brew install ollama

第二步：下载第一个模型

# Llama 3.2 (3B 参数，约 2GB，推荐配置)
ollama pull llama3.2

# 如需更强能力，用 7B (约 4GB)
ollama pull llama3.2:7b

# 中文优化模型 Qwen2.5 (推荐)
ollama pull qwen2.5:7b

# 代码专用模型
ollama pull codellama:7b

第三步：启动并使用

# 启动对话
ollama run llama3.2

# 查看已下载模型
ollama list

# 删除不需要的模型
ollama rm llama3.2:7b

第四步：配置 API 服务（高级）

# 启动 Ollama 服务端（支持 API 调用）
ollama serve

# 默认端口 11434，可通过 curl 调用
curl http://localhost:11434/api/generate -d '{
  "model": "llama3.2",
  "prompt": "用一句话解释量子计算"
}'

第五步：与 OpenClaw 集成

# 在 OpenClaw 中配置本地模型
# 编辑 ~/.openclaw/config.yml 或使用命令

# 设置模型地址
export OLLAMA_BASE_URL=http://localhost:11434

# 之后 OpenClaw 的任务可以调用本地模型

📖 使用指南

日常使用场景

  • 代码助手: 运行 ollama run codellama:7b，粘贴代码片段让它 review 或解释
  • 文档总结: 把长文章粘贴进去，让它提取关键点
  • 写作助手: 用 qwen2.5:7b 处理中文写作，支持续写、润色、翻译
  • 本地知识问答: 配合 RAG 框架（如 RAGFlow）构建私有知识库

模型选择建议 (16GB 内存)

  • Llama 3.2 3B: 速度最快，日常对话首选，内存占用约 6GB
  • Llama 3.2 7B: 能力更强，适合代码和复杂推理，内存占用约 10GB
  • Qwen2.5 7B: 中文能力突出，适合中文写作和对话
  • Phi-2 2.7B: 体积最小，Mac Mini 轻松跑满，适合轻量任务
⚠️ 内存注意: Mac Mini M4 基础款 16GB 内存，同时跑 Ollama + 其他应用可能会吃紧。建议：
  • 同时只跑一个模型
  • 关闭不用的浏览器标签页
  • 不用时执行 ollama stop 释放内存

进阶配置：GPU 加速

Ollama 在 M4 芯片上默认使用 Metal GPU 加速，无需额外配置。如需查看是否生效：

# Mac Mini M4 的 M4 芯片会自动使用 Metal
# 无需手动开启

✨ 方案优势

经济效益

  • 零边际成本: 一次安装，永久使用，无 API 按量计费
  • 对比 ChatGPT Plus: 节省 $20/月 × 12 = $240/年
  • 对比 GPT-4 API: 中等用量下约节省 $50-200/月

隐私安全

  • 数据不出设备: 所有对话在本地处理，敏感信息不上传云端
  • 无审计风险: 公司或医疗等敏感数据无需担心第三方访问
  • 离线可用: 无网络连接时仍可使用（模型已下载）

灵活性

  • 多模型切换: 一个命令换不同模型
  • 开源透明: 模型权重可审计，无闭源风险
  • 定制微调: 可用本地数据微调模型（需更多内存）

💡 专业技巧

  • 模型量化: 使用 .bin 量化版本可大幅减少内存占用，能力损失较小
  • 温度参数: 创意任务用 temperature 0.9，精确任务用 0.3
  • 系统提示词: 在 Modelfile 中设置系统提示词，定制模型行为
  • API 集成: 配合 Cherry Studio 或 OpenWebUI 做图形界面
  • 自动化脚本: 写 shell 脚本一键启动常用模型组合
