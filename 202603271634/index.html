🔧 实施方案

第一步：安装 Obsidian

# 方法1: 官网下载
# 访问 https://obsidian.md/download
# 下载 macOS 版本

# 方法2: Homebrew
brew install --cask obsidian

第二步：创建笔记结构

# 推荐的文件夹结构
vault/
├── 00-Daily/          # 每日笔记
├── 01-Projects/       # 项目笔记
├── 02-Areas/         # 责任区域
├── 03-Resources/     # 资源库
├── 04-Archives/       # 归档
└── z-Templates/      # 笔记模板

第四步：双向链接与图谱

# 创建链接
[[笔记标题]]           # 链接到其他笔记
[[笔记标题#章节]]      # 链接到特定章节
[[笔记标题|显示文本]]  # 使用自定义显示文本

# 效果
# - 按住 Cmd 点击链接跳转到目标笔记
# - 右键"打开反向链接"查看谁引用了这篇
# - 查看本地图谱了解笔记关系网络

📖 使用指南

  • 每日笔记: 每天创建一篇，记录工作学习内容
  • 项目笔记: 每个项目一篇总笔记，链接所有相关内容
  • 标签系统: 用 #标签 组织内容，如 #工作 #学习 #想法
  • 搜索: Cmd+O 快速搜索，Cmd+Shift+F 全局搜索
  • 图表视图: 查看笔记之间的链接关系图

✨ 方案优势

  • 本地优先: 所有数据存在本地，不用担心云服务涨价或宕机
  • 双向链接: 像维基百科一样连接知识，发现隐藏关联
  • 免费强大: 基础功能全部免费，付费功能也只需一次买断 $50
  • 生态丰富: 数百个社区插件，定制化程度极高
  • Markdown 格式: 纯文本存储，长期可读，导出无障碍

⚠️ 常见问题与最佳实践

  • Vault 越来越大: 超过 10,000 笔记后，频繁搜索可能变慢。定期归档 + 使用 Dataview 插件做索引优化。
  • 插件冲突: 安装多个社区插件后有时会冲突。保持 20-30 个活跃插件，定期检查 Safe Mode 排查。
  • 双向链接滥用: 不是所有东西都需要链接。链接应该是概念关联，而非文件夹替代品。
  • 模板太多反而低效: 从 3-5 个核心模板开始，逐渐扩展。
  • AI 隐私: 使用 Obsidian Git 插件配合私仓，确保笔记不泄露。云同步（iCloud/OneDrive）也需注意。
