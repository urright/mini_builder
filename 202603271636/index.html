🔧 实施方案

第一步：准备外置硬盘

# 格式化硬盘为 APFS 或 ext4 (如果只用 Mac 可用 APFS)
# 创建备份目录
mkdir -p /Volumes/BackupMini/snapshots
mkdir -p /Volumes/BackupMini/rsync-excludes.txt

第二步：创建备份脚本

#!/bin/bash
# 保存为 ~/Scripts/rsync-backup.sh

SOURCE="/Users/jeremyren"
DEST="/Volumes/BackupMini/snapshots"
EXCLUDES="/Volumes/BackupMini/rsync-excludes.txt"

# 要排除的文件夹
cat > $EXCLUDES << 'EOF'
.DS_Store
.Trashes
.AppleDouble
Downloads/
Library/Caches/
Library/Logs/
Node_modules/
```

**关键配置**：
- **硬盘**：推荐 1TB+ 外置 SSD，256G Mac mini 存储不够用
- **快照保留**：每小时1个保留24小时，每天1个保留30天，每月1个保留1年
- **加密**：敏感文件可用 `encfs` 加密备份目录

**效果**：
- 首次全量备份后，每次增量仅上传修改的文件
- 占用空间 = 首备份 + (日均变更 × 快照数)
- 100GB 数据 + 30个快照约需 120-150GB

这个方案适合不想付 iCloud 订阅费、需要本地备份且有一定技术基础的用户。配合 Time Machine 使用效果更好——Time Machine 做全量恢复，rsync 做选择性提取。

⚠️ 常见错误与数据安全建议

  • --delete 危险：初次使用前建议先不加 --delete 测试。不小心删掉整个备份的事故很常见。
  • 3-2-1 原则：3 份副本，2 种介质，1 份异地。云端备份（Backblaze）是 rsync 本地备份的重要补充。
  • 256GB 空间紧张：用外置 SSD 做备份，1TB 以上最佳。内置硬盘专门给系统用。
  • 加密备份：敏感数据备份前用 gpg -c 加密，或使用带硬件加密的硬盘盒。
  • 验证备份：每月一次 rsync --dry-run 检查完整性，或定期随机恢复几个文件验证。
