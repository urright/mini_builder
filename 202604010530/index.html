🔀 模型抽象层 (核心)

定义统一接口，让业务代码与具体模型解耦：

1. 模型客户端抽象

# config/models.py
import os
from enum import Enum

class ModelProvider(Enum):
    CLOUD = "cloud"      # 开发: minimax / OpenAI
    LOCAL = "local"       # 交付: Ollama

def get_model_client():
    mode = os.getenv("DEPLOY_MODE", "cloud")
    
    if mode == "local":
        # 交付: 使用本地 Ollama
        from model_clients.ollama_client import OllamaClient
        return OllamaClient(
            base_url=os.getenv("OLLAMA_BASE_URL", "http://localhost:11434"),
            model=os.getenv("OLLAMA_MODEL", "qwen2.5:7b")
        )
    else:
        # 开发: 使用云端 API
        from model_clients.cloud_client import CloudClient
        return CloudClient(
            provider=os.getenv("CLOUD_PROVIDER", "minimax"),
            api_key=os.getenv("CLOUD_API_KEY"),
            model=os.getenv("CLOUD_MODEL", "minimax/m2.7")
        )

2. 统一调用接口

# config/models.py (续)
class ModelClient:
    """统一接口，所有业务代码调用这个"""
    
    def __init__(self):
        self._client = get_model_client()
    
    def complete(self, prompt: str, **kwargs):
        """生成文本"""
        return self._client.complete(prompt, **kwargs)
    
    def chat(self, messages: list, **kwargs):
        """对话"""
        return self._client.chat(messages, **kwargs)
    
    def transcript(self, audio_path: str):
        """语音转文字 (Whisper)"""
        return self._client.transcript(audio_path)

# 全局单例
model = ModelClient()

3. 业务代码完全不感知模型

# src/handlers/email_summary.py
from config.models import model

def summarize_email(email_content: str) -> str:
    """邮件摘要处理 — 不关心用哪个模型"""

    {email_content}
    只返回待办事项，不要其他内容。"""

    return parse_todo_list(response)

🔄 CI/CD 流程 (GitHub Actions)

# .github/workflows/ci.yml
name: CI/CD Pipeline

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  # ====== Job 1: 代码质量检查 ======
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      
      - name: Install dependencies
        run: pip install -r requirements.txt
      
      - name: Lint
        run: |
          pip install flake8 black
          flake8 src/ --max-line-length=120
          black --check src/
      
      - name: Type check
        run: pip install mypy && mypy src/

  # ====== Job 2: 单元测试 (Mock) ======
  unit-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install -r requirements.txt pytest pytest-mock
      - run: pytest tests/test_handlers.py -v --mock

  # ====== Job 3: 集成测试 (云端 API) ======
  integration-test:
    runs-on: ubuntu-latest
    if: github.event_name == 'pull_request'
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install -r requirements.txt
      
      - name: Run with cloud API
        env:
          DEPLOY_MODE: cloud
          CLOUD_API_KEY: ${{ secrets.CLOUD_API_KEY }}
          CLOUD_MODEL: minimax/m2.7
        run: pytest tests/integration/ -v

  # ====== Job 4: 构建 +打包 ======
  build:
    runs-on: ubuntu-latest
    needs: [lint, unit-test]
    if: github.ref == 'refs/heads/main'
    steps:
      - uses: actions/checkout@v4
      
      - name: Build package
        run: |
          mkdir -p dist
          pip install -r requirements.txt --target=dist/package
          cp -r src/ dist/
          cp .env.production dist/.env  # 生产配置 (无云端 key)
          tar -czf my-ai-app.tar.gz dist/
      
      - name: Upload artifact
        uses: actions/upload-artifact@v4
        with:
          name: my-ai-app
          path: my-ai-app.tar.gz

  # ====== Job 5: 交付部署 (触发客户机更新) ======
  deploy:
    runs-on: ubuntu-latest
    needs: build
    if: github.ref == 'refs/heads/main'
    steps:
      - name: Trigger client update
        run: |
          # 方式 A: SSH 到客户机器执行更新脚本
          # 需要提前配置 SSH secrets
          ssh ${{ secrets.CLIENT_HOST }} "
            cd ~/my-ai-app
            ./scripts/update.sh
          "
          
          # 方式 B: 调用客户机的 webhook
          curl -X POST ${{ secrets.CLIENT_WEBHOOK }} \
            -d '{"version": "${{ github.sha }}"}'

交付物

  • my-ai-app.tar.gz — 你的应用包（含本地配置）

客户安装流程 (自动化脚本)

# scripts/install_ollama.sh
#!/bin/bash
set -e

echo "📦 安装 Ollama..."

# 下载安装 Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 安装推荐模型 (qwen2.5 7B, 适合大多数场景)
ollama pull qwen2.5:7b
# 如需更大模型:
# ollama pull qwen2.5:14b

# 验证安装
ollama list

echo "✅ Ollama 安装完成"
echo "访问 http://localhost:11434 确认服务运行"
# scripts/setup_client.sh
#!/bin/bash
set -e

APP_DIR="$HOME/my-ai-app"
OLLAMA_URL="http://localhost:11434"

echo "🔧 客户环境初始化..."

# 1. 检查 Ollama 是否运行
if ! curl -s "$OLLAMA_URL/api/tags" > /dev/null; then
    echo "❌ Ollama 未运行，请先运行 install_ollama.sh"
    exit 1
fi

# 2. 解压应用包
mkdir -p "$APP_DIR"
tar -xzf my-ai-app.tar.gz -C "$APP_DIR"

# 3. 配置生产环境
cp .env.production "$APP_DIR/.env"

# 4. 设置开机自启 (launchd 或 systemd)
# macOS:
cp com.myapp.agent.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.myapp.agent.plist

# 5. 启动服务
cd "$APP_DIR"
./start.sh

echo "✅ 环境初始化完成"
echo "访问 http://localhost:3000 使用应用"

在 GitHub → Settings → Secrets and variables → Actions 中配置：

🔍 常见问题排查

Q: 本地模型输出质量差

可能原因：模型量化太狠或模型太小。

# 检查建议
# 1. 切换更大模型
OLLAMA_MODEL=qwen2.5:14b  # 而非 7b

# 2. 换成更高精度量化
OLLAMA_MODEL=qwen2.5:14b-q4_K_M  # 而非 q4_0

# 3. 调整 prompt，简化任务

Q: 本地模型速度慢

# 1. 检查 M4 Pro 内存占用
ollama ps

# 2. 停止其他大模型
ollama stop qwen2.5:72b  # 只留一个主模型

# 3. 减少上下文长度
model.complete(prompt, options={"num_ctx": 4096})

Q: 客户机 CI 失败

# 客户交付前本地验证
export DEPLOY_MODE=local
pytest tests/ -v

# 确保本地模型兼容性
ollama run qwen2.5:7b "你好"  # 验证模型能正常运行
