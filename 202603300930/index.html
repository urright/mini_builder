Obsidian 第二大脑 + LLM

Mac Mini M4 + Obsidian + 本地 LLM，打造会思考、能关联、零订阅的个人知识系统

Obsidian 知识管理 本地LLM Second Brain Zettelkasten Ollama 可建站

💡 方案简介

在 Mac Mini M4 上运行 Obsidian 作为笔记前端，Ollama 本地 LLM 作为「智能连接引擎」，自动为笔记建立语义关联、生成摘要、回答关于你自己知识库的问题。

所有数据留在本地，零订阅，隐私绝对安全。这是比 Notion AI、ChatGPT 更适合深度知识工作者的方案。

📊 真实案例

Blopig 博客：Obsidian + LLM 功能知识栈（2026）

博主配置了 Obsidian vault 配合本地 LLM（Ollama），实现三个核心自动化：

  1. 自动建立内部链接 — LLM 扫描所有笔记，根据语义自动补充 Obsidian 的 [[双向链接]]，无需手动维护图谱
  2. 知识库问答 — 用自然语言询问整个笔记库，LLM 基于本地知识生成答案，而非依赖云端 API
  3. 摘要生成 — 长篇笔记自动压缩为要点，适合快速复习
零订阅 100%本地 语义搜索

🛠️ 实施步骤

1

安装 Obsidian + 初始化 Vault

# macOS 安装 Obsidian
brew install --cask obsidian

# 或从官网下载：https://obsidian.md/download
# 创建 vault（存放笔记的文件夹）
mkdir ~/Obsidian-Vault
# 首次打开 Obsidian，选择「打开保险库」→「新建保险库」
2

安装 Ollama 并拉取本地模型

# 安装 Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 推荐在 M4 16GB 上运行的模型
ollama pull phi:3.8b          # 最小配置，~3GB，响应快
ollama pull qwen3:4b          # 通用，~4GB，中英文皆可
ollama pull mistral-small-3:latest  # 推荐日常，~4GB，速度最快

# 验证运行
ollama run qwen3:4b "你好"
3

配置 Obsidian + LLM 联动（本地方案）

方案 A：Text Generator 插件（推荐）

# 1. Obsidian 安装社区插件：Settings → Community Plugins → 搜索 "Text Generator"
# 2. 配置本地 Ollama 端点：
#    Plugin Settings → Ollama API URL: http://127.0.0.1:11434
# 3. 设置系统提示词模板：
#    "你是一个知识管理助手，根据以下笔记内容回答用户问题：\n{{CONTENT}}\n\n用户问题：{{SELECTION}}"'

方案 B：Open WebUI 连接 Obsidian vault

# 用 Open WebUI（RAG 增强）访问 Obsidian vault
# 挂载 vault 为文件源，实现语义搜索
# 详细步骤见方案 202603300130（Open WebUI + Ollama）
4

配置自动备份（3-2-1 原则）

# Obsidian Git 插件配置（Community Plugins）
# Settings → Git → 勾选 "Automatic Backup" 和 "Auto backup on save"
# Backup interval: 5-30 分钟

# rsync 定时同步到外置硬盘（可选）
# crontab -e
0 */3 * * * rsync -avz ~/Obsidian-Vault/ /Volumes/Backup/Obsidian/

# Backblaze B2 云端备份（$7/月，无限容量）
# 安装 B2 CLI: brew install b2-cli
b2 sync ~/Obsidian-Vault/ b2://your-bucket/obsidian/
5
