🔧 实施方案（详细版）

第一步：安装 Docker Desktop

# 方法1: Homebrew (推荐)
brew install --cask docker

# 方法2: 官网下载
# 访问 https://docker.com/products/docker-desktop/
# 下载 macOS 版本并安装
# 安装后确保 Docker Desktop 已启动

第二步：创建 n8n 配置目录

mkdir -p ~/.n8n/data
mkdir -p ~/n8n-workflows

# 创建 docker-compose.yml
cat > ~/n8n-workflows/docker-compose.yml << 'EOF'
version: '3'
services:
  n8n:
    image: n8nio/n8n
    container_name: n8n
    ports:
      - "5678:5678"
    volumes:
      - ~/.n8n/data:/home/node/.n8n
      - /Users/jeremyren:/shared
    environment:
      - N8N_DEFAULT_BINARY_DATA_MODE=filesystem
      - N8N_ENCRYPTION_KEY=your-secret-key-here
      - WEBHOOK_URL=http://localhost:5678/
    restart: unless-stopped
EOF

第三步：启动 n8n

cd ~/n8n-workflows
docker-compose up -d

# 查看日志确认启动成功
docker logs -f n8n

# 启动后访问
# http://localhost:5678

第四步：配置外网访问（可选）

# 使用 ngrok 暴露本地 n8n 到公网（用于接收 webhooks）
# 1. 安装 ngrok
brew install ngrok

# 2. 注册并获取 token (免费)
# 访问 https://ngrok.com/

# 3. 配置 token
ngrok config add-authtoken YOUR_TOKEN

# 4. 创建 HTTPS 隧道
ngrok http 5678

# 5. 使用生成的 HTTPS URL 作为 webhook 地址

第五步：常用工作流模板快速导入

# 访问 n8n 工作流模板库
# https://n8n.io/workflows

# 热门模板推荐:
# 1. "Auto-Responder for Telegram Bot" - 自动回复机器人
# 2. "AI Twitter Agent" - AI Twitter 代理
# 3. "Notion to Discord Alert" - Notion 到 Discord 通知
# 4. "Daily AI Summary to Slack" - 每日 AI 摘要推送
# 5. "Website Monitoring with StatusPage" - 网站监控

# 导入方法:
# 1. 在 n8n.io 找到想要的工作流
# 2. 点击 "Copy workflow"
# 3. 在本地 n8n 点击右上角 "Import from JSON"
# 4. 粘贴复制的 JSON

📖 使用指南：5 个真实节省时间的工作流

工作流 1: AI 驱动的销售线索处理

触发: 新 Google Form 提交
  → AI Node (GPT-4): 分类线索质量
  → IF: 质量 = 高
      → 创建 HubSpot 联系人
      → 发送 Slack 通知到 #sales-channel
      → 创建日历事件 (30分钟后跟进)
  → IF: 质量 = 中
      → 添加到 HubSpot 等待队列
      → 发送邮件自动回复
  → IF: 质量 = 低
      → 标记待审阅

效率提升: 手动 15分钟/条 → 自动化 0分钟 + 3分钟跟进
每周节省: ~3-5小时

工作流 2: 每日自动报告生成

触发: 每天 9:00 AM (Schedule Trigger)
  → HTTP Request: 获取昨天 CRM 数据
  → AI Node: 生成摘要 + 关键洞察
  → Slack: 发送报告到 #daily-updates
  → Google Sheets: 追加今日数据行
  → Notion: 更新项目管理数据库

效率提升: 手动 45分钟 → 自动化 30秒
每月节省: ~15小时

工作流 3: 社交媒体自动发布

触发: RSS Feed 新文章 (Trigger)
  → AI Node: 生成 3 条推文版本
  → AI Node: 生成 LinkedIn 长帖
  → Twitter API: 发布推文
  → LinkedIn API: 发布帖子
  → Notion: 记录已发布内容

效率提升: 手动 30分钟/篇 → 自动化 2分钟
每月节省: ~2-4小时

工作流 4: 客户支持工单自动化

触发: Gmail 收到客户邮件
  → AI Node: 判断紧急程度 + 提取关键信息
  → IF: 紧急
      → 创建 Zendesk 工单 (高优先级)
      → 发送 Slack 到 #urgent-support
      → 发送自动回复邮件
  → IF: 普通
      → 创建 Zendesk 工单 (普通)
      → 添加到待处理队列

效率提升: 平均响应时间 4小时 → 30秒
客户满意度提升: ~25%

工作流 5: 监控系统 + 自动告警

触发: 每 5 分钟 (Schedule)
  → HTTP Request: GET 目标网站状态
  → IF: 状态码 != 200
      → Slack: 发送告警到 #incidents
      → 创建 PagerDuty 事件
      → 发送 SMS (Twilio)
      → 记录到 incident-log

效率提升: 人工监控 24/7 → 全自动
MTBF (平均故障恢复时间) 提升: ~60%

⚠️ 常见坑与解决方案

  • Webhook 触发器公网访问: 本地 n8n 需要 ngrok 或 Cloudflare Tunnel 暴露到公网。推荐用 ngrok 免费版，够个人使用。
  • Docker 内存占用: n8n 在 Docker 中默认内存限制较松。编辑 docker-compose.yml 添加 mem_limit: 512mb 防止过度占用。
  • 中文乱码: 在 docker-compose.yml 添加环境变量 LANG=C.UTF-8 和 NODE_ENV=production。
  • 长工作流性能: 超过 50 个节点的工作流建议拆分成多个子工作流，用 Execute Workflow 节点调用。
  • API 频率限制: 调用第三方 API 时添加 Wait 节点控制频率，避免触发限制。
  • 数据备份: 定期备份 ~/.n8n/data 目录，或配置 PostgreSQL 外部数据库。

🆕 2025-2026 n8n 新功能

  • AI Agent 节点: 内置 LangChain 支持，无需额外配置即可使用 AI Agent
  • 多租户支持: 一个实例支持多个用户，适合小团队使用
  • 增强的调试工具: 逐节点执行、变量检查、执行历史回放
  • 云端版本 n8n cloud: 如果不想自托管，也有托管版本可选（但有任务数限制）
  • HLC 加密: 支持端到端加密，数据安全更有保障
  • GitOps 工作流: 工作流可以导出为 JSON 文件，纳入 Git 版本控制
